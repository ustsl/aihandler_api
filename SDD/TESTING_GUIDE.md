# Testing Guide

## Запуск
- Полный прогон: `uv run pytest -q`
- Выборочно: `uv run pytest tests/prompts -q`

## Принципы
- Интеграционные тесты через FastAPI `TestClient`.
- Для AI-вызовов используется mock в `tests/conftest.py`.
- Сеть в тестах не используется.

## Структура тестов по приложениям
- `tests/users/*`: пользователи, токены, настройки, баланс.
- `tests/prompts/*`: CRUD, доступы, валидация, приватность.
- `tests/queries/*`: кеш, история, валидация запроса, story-crop.
- `tests/scenarios/*`: создание/обновление/запуск/ошибки.
- `tests/analytics/*`: сервисная аналитика и авторизация.
- `tests/modules/gpt/*`: модульные тесты расчетов стоимости.

## Фикстуры
- `tests/conftest.py`: DB setup, dependency override, auth headers, mock GPT handler.
- `tests/*/fixtures.py`: предметные фикстуры по доменам.

## Рекомендации при добавлении функционала
1. Добавляйте минимум один позитивный и один негативный тест на новый endpoint.
2. Если меняется бизнес-ветка (баланс, доступ, кеш), добавляйте regression-тест.
3. Если меняется SQL/индекс — добавляйте проверку поведения через интеграционный тест.
