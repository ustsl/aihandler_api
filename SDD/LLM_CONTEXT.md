# LLM Context

## Назначение сервиса
`ai_handler` — FastAPI backend для управления пользователями, промптами, сценариями и выполнением AI-запросов.

## Главные приложения
- `users`: создание пользователя, баланс, токен, настройки, блокировка.
- `prompts`: CRUD промптов с приватностью (`is_open`) и владельцем (`account_id`).
- `queries`: выполнение запросов к AI, кеш похожих запросов, метрики качества ответа.
- `scenarios`: цепочки промптов для последовательной обработки.
- `analytics`: просмотр истории запросов.

## Ключевые инварианты
- Доступ в `users/*` только по `SERVICE_TOKEN`.
- Доступ в `prompts/*`, `queries/*`, `scenarios/*` только по user token конкретного `telegram_id`.
- Баланс пользователя списывается только при ненулевой стоимости запроса.
- Повторный запрос может быть отдан из кеша истории (`cost=0`, `quality_metrics.cached=true`).
- Приватный prompt нельзя читать/обновлять/удалять чужим пользователем.

## Текущая интеграция с AI
- AI вызов выполняется через `src/modules/gpt/*`.
- В тестах внешний AI вызов замокан на уровне `tests/conftest.py` (без сети).

## Что важно при изменениях
- Не ломать текущие API-контракты и URL.
- Если меняются модели БД — добавить Alembic миграцию.
- Любой новый функционал покрывать тестами (интеграционными и/или unit).
- Проверять, что `uv run pytest -q` проходит полностью.
